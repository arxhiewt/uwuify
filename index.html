How it works

Three steps to get started

From invite to active in under a minute.

01

Add uwuify to your server

Invite the bot and grant it the required permissions. The whole process takes under a minute.

02

Set up a channel webhook

Run /webhook #channel in the channel you want uwuify to work in. One webhook per server is supported.

03

Activate on a member

Run /uwuify @user 5m. Every message they send in that channel is intercepted and transformed automatically.

Messages are deleted and re-sent through a webhook so the transformation appears seamless in chat.
Due to Discord API limits, only one active transformation is allowed per server at a time.
Commands

Everything you need to know

/uwuify <user> <duration>Moderator

Activates uwuify on a member for the specified duration. Every message they send in the configured channel is transformed in real time.

e.g. /uwuify @CoolEthan133 5m

/webhook <#channel>Moderator

Sets up the webhook for a channel. Must be run before activating uwuify. Only one webhook per server is allowed — running this again replaces the existing one.

e.g. /webhook #general

/helpAll members

Displays a quick-reference list of commands directly in Discord.

Privacy Policy & Terms

Last updated: July 2026

Privacy Policy

1. What we collect

When your server invites uwuify, we store three things: your server's ID, the webhook ID created by the /webhook command, and — only if the "Create Instant Invite" permission is granted at invite time — a standing invite link to your server. All three are visible only to the bot's developer and are used solely to run and support the bot.

2. What we never collect

uwuify uses zero AI. Message content is never read by a language model, never logged, and never written to a database — it's transformed in memory and discarded the instant it's reposted. We also never store usernames, user IDs, or any other personal data.

3. Why we keep an invite link

A saved invite link lets the developer join your server directly if support is ever needed, e.g. to help fix a broken webhook. If you ever see the developer sending messages in your server, that's what's happening — they've used the invite your server generated.

4. Keeping your server private

If you'd rather not generate a standing invite, untick Create Instant Invite when adding uwuify (or remove it afterwards in Server Settings → Roles). uwuify works exactly the same either way — this permission only affects the developer's ability to rejoin later.

5. Removing your data

Kick uwuify from your server at any time to delete its stored server ID, webhook ID, and invite link.

6. Third parties

uwuify doesn't sell or share data with anyone. What's stored is used exclusively to operate the bot.

7. Contact

Questions? Join our support server and open a thread, or message the developer directly on Discord.
