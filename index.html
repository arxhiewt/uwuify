Troubleshooting & Documentation

Everything you need to fix and understand uwuify.

How does uwuify work?

uwuify uses Discord webhooks to replace user messages in real time. When activated with /uwuify, the bot temporarily takes control of a user's messages.

  • Messages are deleted immediately after being sent
  • They are re-sent using a webhook
  • The text is transformed using the uwu engine

What are uwuify limits?

  • Maximum 2 active users per server
  • Each session lasts 5 minutes
  • 5 second moderator cooldown
  • Messages are queued to avoid Discord rate limits

Webhook error? Transformation not working?

  • Ensure bot has Manage Webhooks permission
  • Run /webhook #channel again
  • Make sure you are in the correct configured channel
  • Check if webhook was deleted manually

Bot could not respond?

  • Bot may be offline or restarting
  • Slash commands may still be syncing
  • Missing permissions in the server
  • Channel restrictions may block interactions

Frequently Asked Issues

uwuify is too slow

This is normal. Messages are processed through a queue system to prevent Discord rate limits.

uwuify is lagging

Lag happens when multiple users are active or Discord API is slow. Reducing active users improves performance.

privacy

Privacy Policy

uwuify only processes messages temporarily for transformation. Messages are deleted immediately after processing.

No permanent storage of messages or user content occurs.
