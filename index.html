how it works

Simple to activate. Instantly hilarious.

Three steps from setup to chaos.

01

Add uwuify to your server

Invite the bot and grant the required permissions. Takes under a minute.

02

Set up a channel webhook

Run /webhook #channel in the channel you want uwuify to operate in. One webhook per server.

03

Activate on a user

Run /uwuify @user 5m and every message they send gets intercepted and re-sent in uwu. They can't escape.

Messages are deleted and re-sent through a webhook so the transformation appears seamless in chat.
Due to Discord API limits, only 1 active transformation is allowed per server at a time.
commands

Everything you need to know.

/uwuify <user> <duration> moderator

Activates uwuify on a user for the specified duration. All messages they send in the configured channel will be intercepted and transformed in real-time.

Example: /uwuify @CoolEthan133 5m

/webhook <#channel> moderator

Sets up the webhook for a specific channel. This must be configured before activating uwuify. Only one webhook is allowed per server — running this again will replace the existing one.

Example: /webhook #general

/help all users

Displays a quick-reference list of commands and links directly in Discord. You can also check the bot's bio for quick links.

Using uwuify

First, invite uwuify to your server. Then run /webhook #channel in the channel you want it to work in. After that, run /uwuify @user duration to activate on someone. That's it.

Run /webhook #new-channel in the new channel. Note: uwuify supports only one webhook per server. Running this command will replace the existing webhook, so the bot will now operate in the new channel instead.

Each transformation session lasts up to 5 minutes by default, or whatever duration you specify when running /uwuify. After the session ends, messages go back to normal automatically.

Run /help in Discord for a quick in-chat reference, or check the bot's bio directly on its profile for links and quick info.

Limitations

Due to current Discord API constraints, uwuify is limited to 1 active transformation per server at a time. This also means only 1 webhook per server is supported. This is a platform limitation we're working within, not a design choice.

A 5-second cooldown is in place to prevent spam and protect against Discord rate limits. This ensures the bot stays stable for everyone in the server.

Troubleshooting
  • Make sure uwuify has the Manage Webhooks permission in your server.
  • Try running /webhook #channel again to recreate the webhook.
  • Confirm you're sending messages in the configured channel.
  • Check if the webhook was deleted manually in Server Settings → Integrations.
  • The bot may be restarting — wait a moment and try again.
  • Slash commands can take a few minutes to sync after the bot joins a new server.
  • Check that uwuify has permission to interact in that channel.
  • Make sure channel restrictions aren't blocking bot interactions.

Messages go through a queue system to prevent hitting Discord's rate limits — a small delay is normal and expected. If lag is significant, it may be due to Discord's API being slow. The bot has no control over Discord-side latency.

Privacy Policy

1. What data we collect

uwuify collects only the minimum data required to function:

  • The Discord server ID of servers that invite the bot, in order to associate webhook configuration with the correct server.
  • An invite record when uwuify is added to a server.

We do not collect, log, or store message content, user IDs, usernames, or any personal user data.

2. How messages are handled

When uwuify is active on a user, messages are intercepted in real-time, transformed, and immediately re-sent via webhook. Messages are processed in memory only and are never written to a database or log file. No permanent copy of any message is retained.

3. Opting out

Server administrators can prevent uwuify from collecting server data by removing the bot from their server. To prevent uwuify from creating webhooks, remove the Create Instant Invite and Manage Webhooks permissions from the bot role in your Server Settings.

4. Third-party services

uwuify operates solely within the Discord platform. We do not share any data with third-party services or advertisers.

5. Contact

For privacy-related questions, join our Discord server and open a support thread.
